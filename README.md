# sorting_legal_articles
# Классификация юридических статей с сайта Pravo.ru
Данный проект посвящен сбору, обработке и классификации юридических статей с сайта Pravo.ru с использованием методов машинного обучения. В рамках проекта реализован полный цикл: от парсинга данных до обучения моделей (Naive Bayes и BERT) и их оценки.

## Описание проекта

Цель проекта — автоматическая классификация юридических статей по тематическим категориям. Исходные данные были собраны с сайта Pravo.ru с помощью парсинга. Затем тексты были очищены, лемматизированы и подготовлены для обучения моделей.

В проекте реализованы два подхода:
- Классический ML: Наивный Байесовский классификатор (MultinomialNB) с TF-IDF векторизацией.
- Глубокое обучение: Fine-tuning модели RuBERT для многоклассовой классификации.

Результаты показывают значительное преимущество BERT-модели, которая достигла accuracy ~86% на тестовой выборке.

## Данные

### Сбор данных

Данные были собраны с сайта Pravo.ru с помощью парсинга (ноутбук pravo_ru_parsing.ipynb):
- Парсинг тематических разделов и ссылок на статьи.
- Сбор текстов статей, заголовков и кратких описаний.
- Очистка от пропусков и дубликатов.

__Итоговый датасет:__ ~18 тысяч статей с тематической разметкой.

### Предобработка

- Приведение к нижнему регистру, удаление пунктуации и лишних пробелов.
- Лемматизация с помощью библиотеки Natasha.
- Удаление стоп-слов.
- Выделение биграмм для улучшения признаков.
- Кодирование целевой переменной:
  - 0: Практика
  - 1: Законодательство
  - 2: Процесс
  - 3: Другое (объединенные редкие категории)

- Разделение данных
  - Обучающая выборка: 70% (12 692 статьи)
  - Валидационная выборка: 15% (2 720 статей)
  - Тестовая выборка: 15% (2 720 статей)

## Модели:

### Naive Bayes (baseline)

Пайплайн:

- TF-IDF векторизация с параметрами:
- max_df=0.8, min_df=3
- max_features=5000
- ngram_range=(1, 2)
- stop_words (русские стоп-слова из NLTK)
- sublinear_tf=True
- Классификатор: MultinomialNB(alpha=1.0)

__Результаты на тесте:__
- Accuracy: 74.85%
- F1 macro: 0.743
- ROC AUC: 0.921

### RuBERT

#### Подготовка:

Минимальная предобработка (lowercase, удаление лишних пробелов и части пунктуации).
Токенизация с помощью AutoTokenizer из библиотеки transformers (модель ai-forever/ruBert-base, максимальная длина 256 токенов).

#### Обучение:

Модель: AutoModelForSequenceClassification с 4 классами.
Гиперпараметры:
- Learning rate: 4e-5
- Batch size: 16 (train), 32 (eval)
- Эпохи: 5 (с early stopping)
- Weight decay: 0.05
- Warmup ratio: 0.06
- Scheduler: cosine
- FP16 (смешанная точность)
- Ранняя остановка (patience=2, порог 0.01).

__Результаты на тесте:__
- Accuracy: 85.81%
- F1 macro: 0.8665
- ROC AUC: 0.9706

RuBERT показал прирост точности +11% по сравнению с классическим подходом, сохраняя высокую обобщающую способность (разрыв между валидацией и тестом минимален).

## Выводы

RuBERT значительно превосходит наивный байесовский классификатор для задачи классификации юридических текстов.
Модель демонстрирует высокую точность (86%) и отличную раделительную способность (ROC AUC > 0.97).
На 4-й эпохе наблюдается переобучение, поэтому оптимальной является модель после 3-й эпохи.
Модель готова к практическому применению для автоматической рубрикации юридических статей.
